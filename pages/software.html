Rsolid

 Rsolid is a R package for normalizing fluorescent intensity data from ABI/SOLiD second generation sequencing platform. It has been observed that the color-calls provided by factory software contain technical artifacts, where the proportions of colors called are extremely variable across sequencing cycles. Rsolid implements a version of quantile normalization algorithm, which transform the intensity values before calling colors. Results show that after normalization, the total number of mappable reads increases by around 5%, and number of perfectly mapped reads increases by 10%. Moreover a 2-5% reduction in overall error rates is observed, with a 2-6% reduction in the rate of valid adjacent color reads.

For more information or download the software, please visit Rsolid software page.

JAMIE

 JAMIE stands for Joint Analysis of Multiple IP Experiments. It is a R package for analyzing ChIP-chip data from multiple experiments. When the experimental conditions are similar, e.g., the transcription factors tend to bind at similar locations, pooling data and analyzing them jointly can improve the sensitivity and specificity of peak finding. JAMIE uses a hierarchical mixture model to capture the locational correlations among data sets and detect peaks from a joint model.

For more information or download the software, please visit JAMIE software page.

makeCGI

 makeCGI is a R package for finding CpG islands (CGIs) from DNA sequences. Basically it fits two HMMs on GC-content and observed to expected CpG counts (O/E), then annotate CGIs based on the posterior probabilities. The main advantages of this approach over the existing methods are (1) the parameters are estimated from data, so that users don't need to provide arbitrary cut-offs (for example, GC-content>50%, O/E>0.6 in current definition); and (2) length of CGI list can be controlled by varying posterior probability cut-off. Results showed that the result from our approach are closer gene promoters and known epigentic marks compared to current definitions.

For more information or download the software, please visit makeCGI software page.

Pseudomarker

 PSEUDOMARKER is a set of programs written in MATLAB for the analysis of QTL data from inbred line crosses. It was originally written by Saunak Sen. I made extensive changes on the codes, rewrote some of the engines in C and wrote a software manual. Some C engine functions were borrowed from Karl Broman's R/qtl package.

PSEUDOMARKER provides a complete workflow for QTL data analysis including data import/export, data quality checking and visualization, genome scans with permutation test, etc. Pseudomarker provides a rich collection of genome scan functions including one, two or multiple QTL genome scans with possible continous or categorical covariates. LOD score threshold may be obtained through permutatin tests (Churchill and Doerge, 1994). One-QTL genome scan can be done under normal or binary model, using EM algorithm (Lander and Botstein, 1989), Haley-Knott regression (Haley and Knott, 1992) and multiple imputation (Sen and Churchill, 2001). Two-QTL can be performed either in exhaustive search (scan all pairs of loci on genome) or using DIRECT method (Lundberg, 2003).

Among the numerous computer programs for QTL mapping, Pseudomarker excels in analyzing date set with a large number of phenotypes. The regression based genome scan algorithm makes the computations on multiple phenotypes very efficient. That makes the analysis on eQTL (expression QTL) data accessible.

For more information on PSEUDOMARKER or to download the software, please visit Here.

R/qtl

 R/qtl is the R version of PSEUDOMARKER (or PSEUDOMARKER is the Matlab version of R/qtl). They basically provide the same functions and share a lot of C codes. R/qtl is mainly written by Karl Broman. I joined Karl since late 2001 on the development of R/qtl and wrote a part of it (mainly on genome scans using multiple imputation method, genome scan permutation tests and some visualization functions).

For more details please visit Karl's R/qtl page.

J/qtl

 J/qtl is a Java GUI for R/qtl. It provides a flexible and powerful working environment for users to perform a variety of tasks.

Although R/qtl is a powerful data analysis software package, it requires some programming skill in R. I developed J/qtl to help the people without programming skill to analyze their data. The software works in such a way that people do their job through the point click interface. The software translates the users' actions into R command, run the command in R, get the result back and display them. So R and R/qtl are the actual work horses. J/qtl uses a third party software Rserve as the middleman between Java and R.

I wrote the software and the manual. Some Java codes were borrowed from TIGR's MeV software. For more details on J/qtl, please visit the J/qtl page in Gary Churchill's group.

R/maanova

 R/maanova is the first R package I've ever written. It turned out to be a success. Katie Kerr and Gary Churchill came up with the idea of analyzing microarray data using an ANOVA approach. They wrote some Matlab scripts and functions to do the calculation. I rewrote those functions and made a R package then wrote the software manual. It's pretty widely used. We have over 2000 unique downloads from our website alone and a very stable user group.

maanova stands for MicroArray ANalysis Of VAriance. It provides a complete work flow for microarray data analysis including:

  • Data quality checks and visualization
  • Data transformation
  • ANOVA model fitting for both fixed and mixed effects models
  • Statistical tests including permutation
  • Confidence interval with bootstrapping
  • Cluster analysis
R/maanova can be applied to any microarray data but it is specially tailored for multiple factor experimental designs. Mixed effects models are implemented to estimate variance components and perform F and t tests for differential expressions.

R/maanova is available under CRAN in here. It's also a part of bioConductor. For detailed information such like installation notes, screen shots, software manual, please visit the R/maanova page in Gary Churchill's group homepage.
